# Morning Entry — January 15, 2026

Starting the day with the starter vault project. The schema files are all done from yesterday, so today is about creating the example entities that demonstrate how everything fits together.

The key insight from the standup with Alex is that the bilateral relations are the hardest part to get right. Both files need to declare the relation — if only one side has it, vault-lint will flag it as a broken bilateral. Going to build the contradicts and trades_off pairs first since those are the bilateral ones.

Feeling good about the overall architecture. The three-layer separation makes it clear what is raw input, what is compiled knowledge, and what is the governing schema. The example entities should make this concrete for anyone looking at the vault for the first time.
